Button Game v1.0

Introduction

This document outlines the functionality and design of the ButtonGamev1 smart contract. The contract is designed to facilitate an onchain game to give away $DEGEN. The last player to press the button wins the pot.

Game Overview

ButtonGamev1 uses multiple onchain variables to determine the game state, game progress, and endgame handling. All game variables are handled by the contract. All gameplay is dependent on confirmed transactions.

Contract Variables

pot: Stores the current value of the game pot.
startBlock: Stores the block number when the game starts.
blockTarget: Stores the target block number to determine the end of the game.
lastBlockPlayed: Stores the block number when the last button press occurred.
freePlaysUsed: Tracks the number of free plays used.
numPlays: Tracks the total number of plays.
daoAddress: Stores the address of the DAO managing the contract.
currentWinner: Stores the address of the current game winner.
freePlayUsed: Tracks whether a player has used a free play.
endGameMet: Indicates whether the end game condition is met.
freePlaysEnabled: Indicates whether free plays are enabled.
playEnabled: Indicates whether the game is enabled.

Gameplay

There are a few key values when it comes to ButtonGame.

1. The Pot: The current contract account balance. This will be transferred to the winner if the block target is reached.

2. The Block Target: The block target is set based on the block number of the last confirmed button press. This means the block target is reset with each button press.

3. The Current Winner: This is the current last player to press the button. If the block target is reached before another confirmed button press is recorded, this player will win the pot.
